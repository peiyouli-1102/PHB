## 阶段 1：初始化项目结构

**目标**：仓库内具备可扩展的静态站点骨架，且与 GitHub Pages 发布方式一致。

**建议目录（可按确认调整）**：

```text
/
├── index.html          # 首页
├── blog/
│   ├── index.html      # 博客列表
│   └── posts/
│       └── numerical-analysis-notes.html   # 第一篇
├── css/
│   └── style.css       # 全局样式（或 split 为 layout.css + blog.css，仍属「无构建」）
├── js/
│   └── main.js         # 必要时少量脚本（导航高亮等）
└── assets/             # 图片与附件
```

**动作**：

1. 创建上述文件占位；`index.html`、`blog/index.html`、第一篇博客页互相可链。
2. 根目录可选：`README.md` 保留简短说明（若课程要求可写本地预览方式）。
3. 首次有意义提交信息写清楚「init static site structure」类说明。

**验证**：本地双击或使用任意静态服务器打开 `index.html`，能点到博客列表和第一篇，附件路径在本地可打开。

**→ 用户确认后继续阶段 2。**

## 阶段 2：完成首页

**目标**：满足 AGENTS 首页清单且风格统一。

**动作**：

1. **固定顶栏**：`<header>` + `<nav>`，链接到首页、博客列表（及可选锚点 `#projects` 等）。
2. **个人简介区块**：姓名 + 教育经历 + 兴趣方向（可用 `<section>` + `<h1>` / `<p>`）。
3. **项目/学习卡片**：2～4 张卡片（`<article>` 或带 heading 的 `<div class="card">`），可含标题、一句描述、外链或占位。
4. **博客入口**：醒目按钮或卡片链到 `blog/index.html`。
5. 全页宽度限制与深色学术风在 `style.css` 中落实。

**验证**：Chrome/Edge 缩小窗口至手机宽度（不必要），导航与卡片不换行崩坏、可读性可接受。

**→ 用户确认后继续阶段 3。**

## 阶段 3：博客列表与第一篇博客页

**目标**：列表仅第一篇可点；文章页结构完整。

**动作**：

1. **博客列表页**：一条列表项，标题、日期（可选）、摘要一行，指向第一篇 HTML。
2. **第一篇博客页**：
   - 标题与元信息（日期、标签可选）。
   - 正文：课程笔记脉络 + **个人理解**段落。
   - **附件区**：课程资料、作业整理 —— 使用 `<a href="..." download>` 或普通外链（同仓相对路径）。
   - **返回主页**：`<a href="/index.html">` 或与 Pages 子路径兼容的相对路径（若站点不在域名根，用相对路径 `../index.html` 更稳）。
3. 通篇语义标签：`main`、`article`、`header`、`footer` 按需使用。

**验证**：从首页 → 列表 → 文章 → 下载/打开每个附件 → 返回主页，路径无 404。

**→ 用户确认后继续阶段 4。**

## 阶段 4：响应式与细节打磨

**目标**：满足「≥手机屏幕」与视觉一致。

**动作**：

1. `meta viewport` 已存在；检查字体字号、行宽、卡片间距在小屏。
2. 固定导航不遮挡首屏标题（可用 `padding-top` 或 `scroll-margin`）。
3. 焦点状态与对比度（深色背景下链接可读）。
4. 全站链接巡检。

**验证**：DevTools 设备模拟 + 真实手机浏览器各走一遍主路径。

**→ 用户确认后继续阶段 5。**

---

## 阶段 5：部署上线（GitHub Pages）

**目标**：公网可访问，与本地链路一致。

**动作**：

1. 推送到 GitHub；在仓库 Settings → Pages 选择分支/目录（与阶段 0 决定一致）。
2. 若站点不在仓库根：修正所有绝对路径（少用 `/css/...`，多用相对路径）。
3. 部署后再次点击附件与返回链接。

**验证**：无痕窗口打开 `https://<user>.github.io/...`，全流程无坏链。

**→ 项目交付；可在 AGENTS.md「当前进度」勾选完成项。**

## 设计决策树（grill-me：分支与推荐）

每行：**决策点** → **推荐**（请不符则改 PLAN 并告知协作方）。

1. **Pages 发布根路径**：项目站 `username.github.io/repo/`  
   → **推荐**：若课程只要求「能访问」，项目仓库 + `main` + `/ (root)` 或 `docs/` 均可；上线前用相对路径避免子路径坑。

2. **附件格式**：可能包含 `.m`、`.tex`、`.pdf` 等（以实际文件为准）  
   → **推荐**：页面与 PLAN **不写死附件个数**；目录内增删文件后，同步更新博客页列表。

3. **样式方案**：单文件 `style.css`
   → **推荐**：单文件减少请求与心智负担，行数过多再拆。

4. **JS 依赖**：少量交互（移动端菜单等）  
   → **推荐**：首版零或极简 JS；导航简单则纯 CSS 亦可。

5. **博客未来扩展**：预留列表模板  
   → **推荐**：静态无构建前提下保持手动一篇一篇加；列表页手动增一行。

6. **中英文**：全中文 
   → **推荐**：与课程交付一致即可，全文中文最省事。
